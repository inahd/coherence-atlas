# 012 — Atlas Control Surface and Operating Manual

Status: proposed

## Purpose
Create a minimal operational surface so Atlas can be run intentionally rather than through dozens of remembered scripts.

## Why now
The repo has reached a scale where the problem is not invention but orientation.
Operators need to see:
- what Atlas is
- how to run it
- how to inspect it
- how to grow it safely

## Deliverables
- one-page Atlas operating manual
- command map
- minimal control surface commands
- README section linking to real workflows

## Candidate commands
- atlas status
- atlas doctor
- atlas build
- atlas visualize
- atlas grow
- atlas research

## Acceptance criteria
- a tired operator can understand the system quickly
- the project no longer feels like a maze of scripts
- Codex and humans share the same operating map
- command docs reflect real entrypoints already in repo

## Notes
This offering is about reducing operator exhaustion and restoring clarity.
