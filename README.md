COHERENCE ATLAS
Local-first, evidence-backed Vedic knowledge graph that grows through automated corpus ingestion — while keeping canon clean.

──────────────────────────────────────────────────────────────────────────────
1) What this is
──────────────────────────────────────────────────────────────────────────────
Coherence Atlas is a workstation toolchain for building a structured “atlas” of
Vedic arts & sciences from real sources (PDFs/text), with a layered truth model:

  Seed (unverified)  →  Auto-proposed (evidence-attached)  →  Canonical (verified)

It is designed to:
- ingest a local research corpus,
- extract text + passage snippets with locators,
- propose structured relations,
- build graphs/dashboards,
- and promote only evidence-backed items into canon.

──────────────────────────────────────────────────────────────────────────────
2) Why it exists
──────────────────────────────────────────────────────────────────────────────
Traditional knowledge systems have strong internal structure (9/12/27/30/108…).
Most digital note systems collapse this into unstructured piles.

Atlas keeps the systemology intact:
- dimensions first (nakshatra, tithi, graha, rashi, padas),
- relations second (deity links, plants, ritual calendar, music praxis),
- evidence always (source_title + locator + excerpt),
- tradition-aware evolution (don’t mix conflicting mappings into mush).

──────────────────────────────────────────────────────────────────────────────
3) Repository layout
──────────────────────────────────────────────────────────────────────────────
datasets/
  Structured entities + relation tables (CSV/YAML). This is the “data spine”.

research/
  Source corpus (PDFs, text). Often kept local; only commit what you intend.

scripts/
  Ingest/extract/stage/safety/report tooling.

memory/
  Generated graphs, reports, indexes (often ignored in git).

prompts/
  Pasteable prompt packs generated from project state.

staging/
  Review queues (CSV previews) for batch adoption into canonical.

──────────────────────────────────────────────────────────────────────────────
4) Data maturity model (how truth is handled)
──────────────────────────────────────────────────────────────────────────────
Seed / Unverified
  - fast population, placeholders allowed
  - may be missing to_id or evidence

Auto-proposed / Attested secondary
  - candidates generated by rules/heuristics
  - must include evidence snippet + locator to be useful

Canonical
  - promoted only when evidence-complete + tradition-labeled
  - gated by validation rules (“guardrails”)

──────────────────────────────────────────────────────────────────────────────
5) Quick start (local)
──────────────────────────────────────────────────────────────────────────────
From /opt/atlas (venv recommended):

  cd /opt/atlas
  source venv/bin/activate
  atlas

Run a full automation pass:

  atlas run

Open dashboard:

  atlas dashboard

Build/open seed graph:

  atlas seedgraph
  atlas graph seed

──────────────────────────────────────────────────────────────────────────────
6) Main commands you’ll use
──────────────────────────────────────────────────────────────────────────────
Core
  atlas run        End-to-end cycle (extract → harvest → evidence → graph → reports)
  atlas factcheck  Hygiene report (missing to_id, missing evidence)
  atlas seedgraph  Rebuild seed graph JSON
  atlas graph seed Open graph viewer

Corpus & evidence
  atlas pdftext / atlas corpus_pdftext    PDF → text
  atlas harvest / atlas corpus_harvest    Passages + tags
  atlas stage N                           Stage candidate relations
  atlas autoapply N MIN_SCORE             Populate relations_auto quickly (non-canonical)

Canonical progression
  atlas promote_auto_export N             Export review sheet
  atlas promote_auto_apply                Apply approved rows to canonical
  atlas guard                             Canonical gate (ontology + evidence checks)

Local API
  atlas api
    /ping, /factcheck, /seedgraph, /files?root=/opt/atlas/research

──────────────────────────────────────────────────────────────────────────────
7) Recommended workflow
──────────────────────────────────────────────────────────────────────────────
Daily (or nightly)
  1) atlas run
  2) atlas factcheck
  3) inspect dashboard + graphs

Weekly
  1) add a small batch of high-quality sources to research/
  2) atlas run
  3) promote only evidence-complete rows into canonical
  4) atlas guard before calling anything “canon”

──────────────────────────────────────────────────────────────────────────────
8) Guardrails / philosophy
──────────────────────────────────────────────────────────────────────────────
- Canonical claims must be evidence-backed.
- Conflicting traditions should be separate, not averaged.
- Aggressive automation is fine, as long as it stays layered.
