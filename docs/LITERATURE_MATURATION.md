# Literature Maturation

Atlas literature should mature rather than endlessly regenerate.

The workflow is:

exploration -> drafting -> refinement -> canonization

## Exploration
Ideas are discussed freely in chat.

## Drafting
Chat discussions are consolidated into structured documents.

## Refinement
Existing documents are improved for clarity, tone, and consistency.

## Canonization
Key documents become stable project literature inside /docs.

## Rule
Once an idea is canonized in the literature, future work should refine or extend it rather than repeatedly rewriting it from scratch.

## Documentation Hierarchy

### Foundational texts
- manifesto
- principles
- philosophy

### Structural texts
- architecture
- layers
- resonance model
- maturation model

### Research texts
- cosmology
- Sanskrit semantics
- pattern bodies
- diaspora research
- symbolic geometry

### Projection texts
- cards
- infographics
- diagrams
- educational media specifications

## Practical Principle

Chat should be used in three modes:
- think
- consolidate
- edit

The repository should remain the long-term memory of the project.
